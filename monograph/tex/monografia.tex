\begin{resumo}

 A linguagem de programação \textit{Scheme} vem sendo utilizada há quase 3
décadas como base para o ensino de programação, conceitos de linguagens de
programação e técnicas de implementações de linguagens de programação. 

A última especificação desta linguagem (\acs{R6RS}), no entanto, causou grande
controvérsia por romper com várias necessidades daqueles que a utilizavam como
plataforma acadêmica. Assim, a comunidade se reúne mais uma vez para ratificar
uma nova especificação (\acs{R7RS}) que possa mais uma vez unificar a direção
que os diversos interesses envolvidos na padronização da linguagem tencionam
seguir.

Este trabalho tem como objetivo demonstrar que a nova versão da linguagem
\textit{Scheme}, em processo de ratificação pelo relatório \acs{R7RS}, define
uma linguagem cuja implementação é passível de ser realizada como objeto de
estudo e experimentação. Isto deve ser possível inclusive por alunos de
graduação em ciência da computação sem a necessidade de uma complexa base
teórica na área de projeto e implementação de interpretadores de linguagens de
programação, utilizando técnicas e conceitos simples que possam ser abordados
dentro do escopo de um curso de ciência da computação sem ser necessária muita
especialização na área.

\end{resumo}

\begin{abstract}

The programming language \textit{Scheme} has been in use for the past 3 decades
as a platform for teaching programming, programming languages concepts and 
programming languages implementation techniques.

The last specification of this language (\acs{R7RS}), though, caused great
controversy by departing from various needs of those who used it as an academic
platform. Therefore, the community gathers once again to ratify a new specification
(\acs{R7RS}) that may once again unify the direction the various interests 
surrounding the standardization of the language intend to follow.

This work attempts to confirm that the new version of the language
\textit{Scheme}, in the process of being ratified by the report \acs{R7RS},
defines a language of which a viable implementation can be made as grounds for
study and experimenting. This should be possible even for undergraduate
students of Computer Science without needing a complex theoretical basis in the
field of design and implementation of programming languages interpreters, using
simple techniques and concepts that could be approached in the scope of a
computer science undergraduate program without specializing too much in this
field.


\end{abstract}
